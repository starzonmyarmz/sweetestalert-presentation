The Sweetest Alert

A <dialog> Migration Story

The Problem

SweetAlert has to go.

  • Bloated bundle — too much for what we needed
  • Constant overrides and per-app customization
  • Inconsistent implementations across PCO apps
  • While accessible, it's not using modern JS APIs that are well-supported
  • Questionable ethics — NSFW sponsors, playing Ukraine's national anthem on Russian IP addresses
The Browser's Gift

The browser already solved this.

Focus trapping

Built-in — no library or manual event management needed.

Keyboard support

Escape to close, tab order — all handled natively.

::backdrop

Native overlay — no extra DOM, no z-index battles.

top-layer promotion

Dialogs always render above everything. Stacking context is someone else's problem now.

What We Built

Sweetest Alert

  • Native <dialog> — way less code
  • Built for PCO — Tapestry tokens, our standards
  • Rails support via Stimulus + DOMPurify
Before
// SweetAlert2 Swal.fire({ title: "Archive group?", icon: "warning", showCancelButton: true, confirmButtonColor: "#3085d6", confirmButtonText: "Archive", })
After
// SweetestAlert SweetestAlert({ title: "Archive group?", icon: "warning", confirmButtonText: "Archive", })
The Strategy

Keep it simple. Ship it early.

The temptation
  • Big rewrite
  • Migration plan doc
  • All or nothing
Our approach
  • Inventory → POC
  • 2–3 instances at a time
  • Expand as we go
How It Happened

The process

  1. Inventory — every swal instance in Accounts
  2. Compare against the Figma spec
  3. POC in one low-traffic area accounts#4080
  4. Convert 2–3 at a time, growing in visibility and complexity
  5. Expand features and tests as each wave reveals new needs
Packaging & Rails Support

Extract. Support. Ship.

  • Extract to an NPM package with docs & demos
  • Switch Accounts over to the package
  • Rails: data-* attrs forwarded to the React component via Stimulus + a helper
# app/helpers/sweetest_alert_helper.rb def sweetest_alert_tag(**opts) tag.div(data: { controller: "sweetest-alert", **opts }) end
